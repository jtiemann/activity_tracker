Chart Debugging Guide

Important: A completely rewritten version of the charts.js file has been installed with enhanced debugging capabilities. This will help diagnose and fix the weekly chart issue.

Step 1: Restart Your Server

First, restart your Activity Tracker server to load the new debug version:

ctrl+c (to stop the server)
npm start (to restart it)

Step 2: Access Debug Functions

After logging in and selecting the "Pull-ups" activity, open your browser's console (F12) and use these debug commands:

Analyze Log Data

window.chartDebug.analyzeLogData();

This will show detailed information about your log data structure and types.

Test Weekly Data Generation

window.chartDebug.testWeeklyData();

This will run the weekly data preparation function and show the results.

Show Raw Data

window.chartDebug.showRawData();

This will display the raw log data loaded in the charts.

View Weekly Chart

window.switchToChart('weekly');

This will switch to the weekly chart view.

Step 3: Check for Errors

After running the debug commands, check for any error messages or warnings in the console.

Step 4: Restore Original File (If Needed)

If you need to restore your original charts.js file, run this command in your project directory:

node restore-charts.js

(This script will be created if you request it)

Common Issues & Solutions

  • String Concatenation: If JavaScript is treating numbers as strings, it will concatenate them instead of adding them numerically.
  • Invalid Dates: If any log dates can't be parsed, they will be skipped.
  • Missing Chart Library: Ensure Chart.js is properly loaded.
  • Canvas Element Issues: The chart canvas element must be correctly created in the DOM.
